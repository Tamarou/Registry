## Executive Summary

The program-creation workflow step classes have two critical bugs where returning `{}` instead of `{stay => 1}` causes the workflow engine to silently skip steps. The HTMX integration in the controller is functional but has inconsistent stash variables across the three fragment render paths, and doesn't handle completion/continuation for HTMX requests. Two pre-existing security issues (hostname-based secret, unauthenticated tenant switching) were also flagged.

## Critical Issues

### [C1] RequirementsAndPatterns::process silently skips step on empty form
- **File:** `lib/Registry/DAO/WorkflowSteps/RequirementsAndPatterns.pm:13`
- **Bug:** `return {} unless exists $form_data->{min_age}` causes the engine to advance `latest_step_id` without user input, skipping the entire step
- **Impact:** A POST with missing/wrong form fields bypasses all requirements validation
- **Suggested fix:** `return { stay => 1 } unless exists $form_data->{min_age}`
- **Confidence:** High
- **Found by:** Logic & Correctness, Error Handling, Contract & Integration, Verifier

### [C2] ReviewAndCreate::process silently skips step when confirm absent
- **File:** `lib/Registry/DAO/WorkflowSteps/ReviewAndCreate.pm:15`
- **Bug:** `return {} unless $form_data->{confirm}` causes the engine to advance past review without creating a Project
- **Impact:** Workflow completes successfully but no Project record is created
- **Suggested fix:** `return { stay => 1 } unless $form_data->{confirm}`
- **Confidence:** High
- **Found by:** Logic & Correctness, Error Handling, Contract & Integration, Verifier

## Important Issues

### [I1] HTMX advance path missing stash variables
- **File:** `lib/Registry/Controller/Workflows.pm:395-405`
- **Bug:** HTMX advance render missing `data_json`, `errors_json`, `workflow_progress` that stay path and GET path provide
- **Impact:** Templates referencing these variables get undef in HTMX advance responses
- **Suggested fix:** Add missing stash keys to match the stay path pattern
- **Confidence:** High
- **Found by:** Logic & Correctness, Contract & Integration, Code Quality

### [I2] Unguarded $next->slug dereference when next_step returns undef
- **File:** `lib/Registry/Controller/Workflows.pm:386-387`
- **Bug:** `next_step` can return undef if workflow step chain is broken, but `$next->slug` is called without nil check
- **Impact:** 500 crash on malformed workflow step graphs
- **Suggested fix:** Guard with `unless ($next)` error render
- **Confidence:** High
- **Found by:** Error Handling, Verifier

### [I3] Project->create can throw uncaught exception
- **File:** `lib/Registry/DAO/WorkflowSteps/ReviewAndCreate.pm:21-31`
- **Bug:** DB constraint violations (duplicate name, null name) cause unhandled die, not caught by `unless ($project)` check
- **Impact:** 500 error instead of user-friendly validation message
- **Suggested fix:** Wrap in eval, return errors on failure
- **Confidence:** High
- **Found by:** Error Handling, Verifier

### [I4] HTMX stay path uses $result->{template_data} instead of prepare_template_data
- **File:** `lib/Registry/Controller/Workflows.pm:352`
- **Bug:** ProgramTypeSelection returns `{stay => 1}` with no template_data key; stay path renders with empty hash instead of calling prepare_template_data
- **Impact:** Stay renders may be missing required template variables
- **Suggested fix:** Always call prepare_template_data as base, merge result template_data on top
- **Confidence:** Medium
- **Found by:** Code Quality
